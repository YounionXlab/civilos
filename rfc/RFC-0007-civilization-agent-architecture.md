# RFC-0007: Civilization Agent Architecture v0.1

## Purpose

Define the architecture of all intelligent entities inside CivilOS.

The objective is not to create NPCs.
The objective is to create evolving civilization participants.

---

# 1. First Principle

Every civilization is composed of agents operating at multiple scales.

CivilOS models four primary agent types.

```
Civilization
    ↑
Organization
    ↑
Individual
    ↑
Environment
```

Each level influences every other level.

---

# 2. Individual Agent

Represents a single intelligent being.

Core attributes:
- Identity
- Personality
- Needs
- Skills
- Memory
- Values
- Relationships
- Goals

Individual agents learn from experience and change over time.

---

# 3. Organization Agent

Examples:
- Government
- Company
- Research Institute
- School
- Family

Organizations possess:
- Shared objectives
- Resources
- Internal rules
- Collective memory
- Reputation

Organizations evolve independently from individuals.

---

# 4. Environment Agent

The world is an active participant.

Examples:
- Climate
- Ecosystems
- Resource systems
- Planetary events
- Disease dynamics

Environment continuously generates new conditions and problems.

---

# 5. Civilization Agent

Represents civilization as an emergent higher-level intelligence.

Properties:
- Cultural identity
- Technological level
- Governance capability
- Collective wisdom
- Historical memory
- CQ metrics

Civilization itself becomes a long-lived adaptive system.

---

# 6. Interaction Graph

Environment ↔ Individual
Individual ↔ Organization
Organization ↔ Civilization
Civilization ↔ Environment

Feedback exists across all layers.

---

# 7. Design Goal

Players should not feel surrounded by scripted NPCs.

They should feel they are participating in a living civilization where every agent can learn, remember and influence history.

---

# Core Principle

The smallest playable unit is not a character.

It is an evolving agent.
